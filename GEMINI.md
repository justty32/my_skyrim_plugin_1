# Gemini Project Context: TemplatePlugin (SKSE Plugin)

This project is an SKSE (Skyrim Script Extender) plugin template for Skyrim SE/AE/GOG/VR, designed for cross-version compatibility using `CommonLibSSE-NG`. It features a specialized build infrastructure for static CRT linkage, automated MO2 packaging, and CI integration.

## Project Overview

- **Type:** C++23 SKSE Plugin
- **Target Games:** Skyrim Special Edition, Anniversary Edition (incl. 1.6.1170), GOG, and VR.
- **Base Library:** [CommonLibSSE-NG](https://github.com/CharmedBaryon/CommonLibSSE-NG)
- **Key Features:**
  - **Static CRT Build:** Produces a standalone `.dll` with no dependencies on `MSVCP140.dll` / `VCRUNTIME140.dll` (ideal for Linux/Proton).
  - **MO2 Packaging:** `scripts/pack.ps1` creates an installable ZIP for Mod Organizer 2.
  - **CI/CD:** GitHub Actions workflow builds on Windows, caches dependencies, and uploads artifacts.
  - **Trace-Level Logging:** Immediate flush logging setup in `src/log.h` for easier debugging under Proton.

## Building and Running

### Prerequisites
- Windows 10/11 with Visual Studio 2022 ("Desktop development with C++" workload).
- `vcpkg` installed with `VCPKG_ROOT` environment variable set.
- (Optional) `SKYRIM_FOLDER` or `SKYRIM_MODS_FOLDER` for post-build auto-deployment.

### Commands (PowerShell)
- **Configure (Release):** `cmake --preset build-release-msvc`
- **Build (Release):** `cmake --build build/release-msvc`
- **Clean Rebuild:** `Remove-Item -Recurse -Force build; cmake --preset build-release-msvc; cmake --build build/release-msvc`
- **Package for MO2:** `scripts/pack.ps1` (creates ZIP in `dist/`)
- **Verify Static Linkage:** `dumpbin /dependents build/release-msvc/TemplatePlugin.dll` (Look for absence of `MSVCP140.dll`).

### Deployment
- The DLL must be placed in `Data/SKSE/Plugins/` within the Skyrim directory or a mod manager's virtual folder.
- If `SKYRIM_FOLDER` or `SKYRIM_MODS_FOLDER` is set, CMake will copy the DLL automatically after building.

## Development Conventions

### Code Structure
- **Entry Point:** `src/plugin.cpp` contains `SKSEPluginLoad`. Initialize handlers in `MessageHandler` (e.g., `kDataLoaded`).
- **PCH:** `src/PCH.h` is mandatory. It includes `RE/Skyrim.h` and `SKSE/SKSE.h`.
- **Utilities:** `src/util.h` contains highly reusable helper structs for pointer math, string manipulation, form parsing, and game object transformation.
- **Logging:** Use `SKSE::log::info/trace/debug/warn/error`. Configured in `src/log.h`.

### Adding Files
- **DO NOT** use globbing in `CMakeLists.txt`.
- Register new `.cpp` files in `cmake/sourcelist.cmake`.
- Register new `.h` files in `cmake/headerlist.cmake`.

### Static CRT Requirements
- If changing CRT linkage (Static vs Dynamic), you **must** update both `CMakePresets.json` (`CMAKE_MSVC_RUNTIME_LIBRARY`) and `cmake/x64-windows-skse.cmake` (`VCPKG_CRT_LINKAGE`).
- Always delete the `build/` directory after changing CRT settings to clear the vcpkg cache.

### SKSE Plugin Info
- `SKSEPluginInfo` is automatically generated by `add_commonlibsse_plugin` in `CMakeLists.txt`.
- **DO NOT** manually add `SKSEPluginInfo(...)` to your source code; it will cause `LNK2005` duplicate symbol errors.
- Metadata (Author, Version, etc.) should be edited in `CMakeLists.txt`.

## Debugging Workflow (Linux/Proton Focus)
Since direct debugger attachment is difficult under Proton, rely heavily on trace logs:
- **Log Path:** `<Proton-Prefix>/drive_c/users/steamuser/Documents/My Games/Skyrim Special Edition/SKSE/TemplatePlugin.log`
- All logs flush immediately on write (configured in `SetupLog()`).

## Key Files Summary
- `CMakeLists.txt`: Project configuration and plugin metadata.
- `CMakePresets.json`: Build configurations and compiler flags.
- `vcpkg.json`: Library dependencies.
- `src/plugin.cpp`: SKSE lifecycle management.
- `src/util.h`: Essential modding helpers (forms, math, nodes).
- `PITFALLS.md`: Log of common build/link errors and their fixes.
- `BUILD.md`: Detailed build instructions for PowerShell/bash.
